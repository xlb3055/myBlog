Springboot 的常用注解（持续更新。。。）
1. @RestController：@Controller 和 @ResponseBody  的组合注解。每个方法的返回值都会以 JSON 或 XML 的形式直接写入 HTTP 响应体中，相当于在每个方法上都添加了 @ResponseBody 注解。
2. @Controller：适用于传统的 MVC 架构，它负责处理请求并返回相应的视图。（@RestController下的方法默认返回的是数据格式，@Controller注解标注的类下面的方法默认返回的就是以视图为格式）
3. @RequestMapping：将请求和处理请求的控制器方法关联起来，建立映射关系。（说白了就是定义一个请求到哪的路径）
4. @reponseBody：用于将controller的返回结果经过转化转为json格式
5. @pathvariable：获取get请求中的参数
6. @GetMapping：用于细化接口的请求类型
7. @PostMapping：用于细化接口的请求类型
8. @SpringbootApplication：用于标记主配置类，其中包含了@SpringBootConfiguration、@EnableAutoConfiguration、@ComponentScan三个注解
9. @SpringBootConfiguration：表示这是一个 Spring 配置类，它等价于 @Configuration 注解，用于声明类作为 Spring 的配置类。
10. @EnableAutoConfiguration：启用 Spring Boot 的自动配置功能，Spring Boot 会根据添加到项目中的依赖库来自动配置应用程序，它可以帮助减少显式的配置代码，因为 Spring Boot 会自动检测和配置合适的 Bean。
11. @ComponentScan：用于指定要扫描的包，以便 Spring 可以在这些包中查找带有 @Component、@Service、@Repository 和 @Controller 等注解的类。默认情况下，@ComponentScan 会在包含 @SpringBootApplication 注解的类所在的包及其子包中进行扫描。
12. @Autoware: 用于实现依赖注入,允许 Spring 自动地将匹配的 Bean 注入到类的字段、构造函数或方法中。
13. @Resource： Java 的一个注解，用于实现依赖注入，和@Autoware类似，但与之不同的是他默认通过name匹配来注入Bean，如果找不到匹配的name则会退回到通过类型匹配。

数据库事务

数据库事务的特性

  1. 原子性：一个事务包含的行为要么全部成功，要么全部不成功，如果在执行中间发生错误，则会进行事务回滚，回滚到事务开始之前的状态
  2. 一致性：事务执行完毕后不会对数据库的完整性造成破坏
  3. 事务的隔离：由于数据库的事务支持多线程，事务隔离是为了防止多个线程的事务造成数据不一致 持久性：事务完成后，对数据的修改是永久的，哪怕系统故障也不会消失

数据库事务的隔离级别

  1. 未提交读：可以读到其他线程修改但没提交的数据，如果那个线程产生错误从而导致回滚，那么这次读取的数据就是脏数据
  2. 提交读：事务在读取数据的过程中，有其他线程事务修改了数据，造成前后数据不一致
  3. 重复读：一开始读取数据为空，但是尝试修改这条不存在的数据却成功了，此时再读取该数据又可以读取到了
  4. 串行读：是最严格的隔离级别。严格按照先后次序执行，但会造成效率大大下降

Java中线程的状态
  1. 新建状态：线程刚刚创建
  2. 就绪/运行状态
  3. blocked状态：未持有锁资源的线程进入阻塞状态
  4. waited状态：wait（）
  5. timed_waited状态：sleep（）
  6. 结束状态

sleep 和 wait 的区别

  1. Sleep使线程进入time_waited状态，经过一定时间自动唤醒；waited是在持有锁时主动释放锁资源进入waited状态，等待手动唤醒。
  2. wait只有在持有锁的时候才能使用，而sleep持有或不持有锁都可以使用。
  3. 在持有锁时使用sleep不会释放锁资源，使用wait会释放锁资源

Java自定义异常
1. 新建一个类，继承Exception
public class MyException extends Exception{
}
2. 添加一个构造方法，和重写一个tostring方法即可
完全体示例：
/**
 * 自定义异常
 */
public class MyException extends Exception{

    private int a;

    public MyException(int a) {
        this.a = a;
    }

    @Override
    public String toString() {
        return "输入数字的范围应当小于等于10，当前为："+a+"。请重新输入";
    }
}

3. 使用：
class Test2{
    public static void main(String[] args) {
        int a = 11;
        try {// try监控区域
            zdyExceptionTest(a);
        } catch (MyException e) {// 想要捕获的异常类型,如果监控区域出现指定异常，则执行以下代码
            System.out.println(e);
        } finally {// 不管抛不抛出异常都执行,一般用来关闭io流
            System.out.println("处理了异常");
        }
    }

    // throws是在方法上抛出异常，在调用这个方法的时候捕获异常
    public static void zdyExceptionTest(int a) throws MyException {
            if (a >= 10) {// 设置如果a>10就手动抛出异常
                throw new MyException(a);// 抛出异常
            }
    }
}

  1. （CORS）使用@CrossOrigen注解配置
  2. 响应报文中加入一个Access-Control-Allow-Origin 的header
  3. （CORS）实现WebMVCConfigurer接口然后重写addCorsMapping方法解决跨域问题
  4. 前端使用代理服务器，如Node.js中使用http-proxy-middleware模块。
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        // 设置允许跨域的路径
        registry.addMapping("/**")
                // 设置允许跨域请求的域名
                .allowedOrigins("http://localhost:8099")
                // 是否允许证书不失效
                .allowCredentials(true)
                // 设置允许的方法
                .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS")
                // 设置允许的头信息
                .allowedHeaders("*")
                // 预检请求有效期，单位为秒
                .maxAge(3600);
    }

Java递归思想

什么是递归？

递归就是自己调用自己！！！！！！

  • 递归思想通常把一个大型复杂的问题层层转化为一个与原问题相似的规模较小的问题来求解。
  • 包括两个部分：
    • 递归头：什么时候不调用自身方法，如果没有头，将陷入死循环
    • 递归体：什么时候需要调用自身方法
    /*
        阶乘
        思路： 当 n == 1 时，返回n（递归到头，该开始层层往回return了）
              当 n != 1 时，做乘法，同时层层递归，相当于f(n) * f(n-1)，然后f（n-1）里面是f(n-1) * f(n-2)
              所以最后相当于 f(n) * f(n-1) * f(n-2) * ... * f(1)
     */
    public static int factorial(int n) {
        if(n == 1) {
            return n;
        } else {
            return n * factorial(n-1);
        }
    }
